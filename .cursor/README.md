# Cursor 전역 설정 디렉토리

이 디렉토리는 **Cursor IDE의 전역(Global) 설정**을 관리합니다.
모든 프로젝트에서 공유되는 설정이 여기에 저장됩니다.

## 디렉토리 구조

```
~/.cursor/
├── mcp.json          # MCP 서버 설정 (전역)
├── ide_state.json    # IDE 상태 정보 (전역)
├── projects/         # 프로젝트별 캐시 데이터
│   └── home-ubuntu/
│       └── mcp-cache.json
└── README.md         # 이 파일
```

## 주요 파일 설명

### `mcp.json`
- **역할**: Model Context Protocol (MCP) 서버 설정
- **범위**: 모든 프로젝트에서 공유되는 전역 설정
- **내용**: 
  - AWS Documentation MCP 서버
  - 브라우저 확장 기능 MCP 서버 (설정 시)
  - 기타 전역 MCP 서버들

### `ide_state.json`
- **역할**: Cursor IDE의 상태 정보 저장
- **범위**: 전역 IDE 상태 (최근 열어본 파일 등)
- **자동 관리**: Cursor가 자동으로 업데이트

### `projects/`
- **역할**: 프로젝트별 캐시 데이터 저장
- **범위**: 각 프로젝트의 캐시 정보
- **자동 관리**: Cursor가 자동으로 생성/관리

## 프로젝트별 설정과의 관계

### 전역 설정 (`~/.cursor/`)
- 모든 프로젝트에서 공유
- MCP 서버 설정
- IDE 전역 상태

### 프로젝트별 설정 (`프로젝트/.cursor/`)
- 특정 프로젝트에만 적용
- 프로젝트 규칙 (rules/)
- 프로젝트 명령어 (commands/)
- 프로젝트별 가이드 문서

## 설정 우선순위

1. **프로젝트별 설정**이 전역 설정보다 우선합니다
2. MCP 서버는 전역 설정에서 관리됩니다
3. 프로젝트 규칙과 명령어는 프로젝트별 설정에서 관리됩니다

## MCP 서버 설정

현재 설정된 MCP 서버:
- ✅ AWS Documentation Server

추가 가능한 MCP 서버:
- 브라우저 확장 기능 (Cursor 확장 기능으로 제공)
- 기타 MCP 서버들

**자세한 설정 방법**: `MCP_SETUP.md` 파일 참조

## 참고사항

- 이 디렉토리는 **전역 설정**이므로 모든 프로젝트에 영향을 줍니다
- 프로젝트별 설정은 각 프로젝트의 `.cursor/` 폴더를 참조하세요
- MCP 서버 설정을 변경하면 Cursor를 재시작해야 할 수 있습니다
